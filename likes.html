Don't dress things up, or try to explain them; just write them down, in a list, in the plainest language possible.

Nick Hornby, High Fidelity

If you've read the book High Fidelity then you know that the protagonist, Rob, is obsessed with making lists of things. I can relate. There are a ton of great tools and resources available to software developers and putting stuff in lists helps make sense of it all. To that end, I've curated a few lists of things that I like and will periodically update them as my likes change.

Tools

Libraries / NuGet Packages

  • AttributeRouting
    Define your routes using attributes on action methods in ASP.NET MVC and Web API. Probably less relevant now that MVC 5 includes it's own attribute routing, but still valuable for legacy projects.
  • AutoMapper
    Automatic mapping between objects (I.e., domain models to view models). Also highly configurable to meet many different use cases.
  • DelegateDecompiler
    Converts computed entity properties for use on the server. See my blog post on the topic.
  • dotless
    A LESS CSS parser for C#.
  • Entity Framework
    I've tried a number of ORMs over the years, and my favorite by far has been Entity Framework. The EF team also seems highly committed to delivering regular improvements, which is always a positive.
  • Glimpse
    Real time diagnostics & insights of web applications. I can't even count the number of debugging hours Glimpse has saved me.
  • HtmlAgilityPack
    This is an oldie but a goodie and hasn't been updated in a number of years, but it still works well and is still the best/only way to work with an HTML DOM tree in code.
  • Humanizer
    Presenting information is one of the most important aspects of most applications, and this library makes it easy to manipulate and display strings, enums, dates, times, timespans, numbers and quantities.
  • Json.NET
    By far the best way of working with JSON in .NET.
  • RazorEngine
    I keep finding little use cases for this library that enables using Razor as a templating engine.
  • T4MVC
    I hate magic strings, and T4MVC is so embedded into my process now I literally can't imaging writing a full MVC application without it.
  • Unity
    I don't always use an IoC container, but when I do, it's (usually) Unity.

Visual Studio Extensions

  • JustCode
    I've used JustCode and ReSharper and like both. Whichever one you choose, they both save countless hours of development time.
  • Productivity Power Tools 2013 (or 2012)
    All the stuff that didn't quite make it into Visual Studio from the folks who created it.
  • Spell Checker
    Visual Studio should have a built-in spell checker, but since it doesn't this fits the bill nicely.

Front End (JavaScript & CSS)

  • Bootstrap
    I am in love with Bootstrap. I want to marry it and have it's babies.
  • jQuery
    The JavaScript library wars are over and jQuery won. While it's certainly possible to build a highly interactive web site or application without it, I certainly wouldn't want to.
  • Kendo UI
    A very good set of pure JavaScript/HTML5 UI components. Has both a free version and a more complete commercial one. Combined with the ASP.NET MVC server-side wrappers, it makes developing rich HTML5 applications in MVC much, much easier.
  • Modernizr
    Browser feature detection with a large set of supported polyfills.
  • Google Prettify
    A lightweight syntax highlighter. I use it on this site.

Development Tools

  • Fiddler
    Allows you to see all the little details of what's going on between your browser and the server (including a local development server).
  • JustDecompile
    A decompiler is a must for any serious .NET toolbox. I've previously used /NET Reflector and really liked it, but JustDecompile is free and it gets the job done just as well.
  • LINQPad
    Lets you query databases using LINQ, write little scripts, experiment with code without having to load the whole Visual Studio environment, create a project, etc.
  • JIRA, Confluence, and Stash
    This is the trifecta of development project management and source control.
  • Loupe
    Real-time centralized log gathering and reporting.
  • scriptcs
    A very light-weight C# scripting engine. Why use PowerShell when you can use C#?
  • SmartGit
    I've tried a bunch of Git clients, and this one is the best IMO.

Utilities

  • Greenshot
    This little utility sits in your system tray and lets you take screen shots by hitting PrtScn. Then you can copy the screen shots to the clipboard, open them in an image editor, or do any number of other things to them.
  • Notepad++
    When Visual Studio is way too much and Notepad is too little.
  • Wireshark
    The free (and open-source) packet sniffer and analyzer.
  • WinSplit Revolution
    This great free Windows layout manager has been discontinued by the developer, but it can still be downloaded.
